# Dashboard Sections Reference

## Section Specifications

### System Status Section

**Purpose:** High-level overview of system state

**Required fields:**
- Vault location (absolute path)
- Watchers running (count or status)
- Pending tasks count
- Completed today count

**Example:**
```markdown
## System Status

**Vault Location:** `C:\Users\...\AI_Employee_Vault`
**Watchers Running:** 1 (Filesystem Watcher)
**Pending Tasks:** 3
**Completed Today:** 7
```

### Quick Stats Table

**Purpose:** Key metrics at a glance

**Required metrics:**
- Tasks in Needs_Action

- Completed (Today)
- Completed (This Week)

```markdown

| Pending Approval | 1 |
```

### Recent Activity Section

**Purpose:** Log of latest actions taken

**Format:** Table with columns:
- Timestamp (YYYY-MM-DD HH:MM:SS)
- Action (action type)
- Status (success/failed/pending)
- Details (brief description)

**Limit:** Last 10 actions

**Example:**
```markdown
| Timestamp | Action | Status | Details |
|-----------|--------|--------|---------|
| 2026-01-11 14:30:22 | task_processed | success | FILE_document.md |
| 2026-01-11 14:25:18 | watcher_detected | success | New file in Inbox |
| 2026-01-11 14:20:00 | dashboard_updated | success | Status refresh |
```

## Update Triggers

Dashboard should be updated:
1. After task processing completes
2. When user requests "update dashboard"
3. On scheduled intervals (e.g., hourly)
4. After critical system events
5. Before generating reports

## Performance Considerations

**Keep updates fast:**
- Cache folder counts (don't recount every second)
- Limit log parsing to most recent file
- Use file count (not content scanning) where possible
- Update asynchronously when possible

**Avoid:**
- Reading all files in /Done (filter by date)
- Parsing every log entry (limit to last N)
- Expensive calculations on every update

## Customization

Users may want to add:
- Custom metrics specific to their workflow
- Business-specific KPIs
- Integration with external services
- Time-based trends (charts, graphs)
- Budget tracking
- Client-specific sections

## Error Handling

If data collection fails:
- Show "N/A" for missing data
- Log the error
- Continue with partial update
- Add alert about data collection issue

**Example:**
```markdown
| Metric | Count |
|--------|-------|
| Tasks in Needs_Action | N/A (Error reading folder) |
```
